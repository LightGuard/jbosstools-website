JBoss Tools and Red Hat JBoss Developer Studio downloads

JBoss Tools 4.1, 4.0, 3.3 and 3.2 are available in both the FREE and Portfolio Editions of Red Hat JBoss Developer Studio 7.x, 6.x, 5.x and 4.x, respectively. Red Hat JBoss Developer Studio, Portfolio Edition comes bundled with Red Hat JBoss Enterprise Application Platform (JBoss EAP). If you want an easy to install and fully configured development environment, try it out. Future development milestones will be available FREE as part of the JBoss Developer Studio Early Access Program.

JBoss Tools is now splitted into two separate packages:

  • the Core version, which contains plugins for CDI, Hibernate, Web Services, OpenShift, AS, Central, Mobile, LiveReload, etc.
  • the Integration Stack (formerly SOA) version, which provides more specific plugins for BRMS (JBPM, Drools, etc.)

The Eclipse Marketplace provides a way to install stable releases of JBoss Tools and Red Hat JBoss Developer Studio from within Eclipse. You can either drag the install button into your Eclipse workspace or click on it to be brought to the Eclipse Marketplace.

To install Stable Releases of JBoss Tools, start up Eclipse, then do:

Help > Install New Software... >

Depending on the version of Eclipse you have or the version of JBoss Tools you want to install, copy the URL from the table below into Eclipse, and hit Enter.

JBoss Tools Core 4.1 (Eclipse Kepler 4.3) http://download.jboss.org/jbosstools/updates/stable/kepler/
JBoss Tools Core 4.0 (Eclipse Juno 4.2.2) http://download.jboss.org/jbosstools/updates/stable/juno/
JBoss Tools Core 3.3 (Eclipse Indigo 3.7.2) http://download.jboss.org/jbosstools/updates/stable/indigo/
JBoss Tools SOA 3.3 (Eclipse Indigo 3.7.2) http://download.jboss.org/jbosstools/updates/stable/indigo/soa-tooling/

When the site loads, select the features to install, or click the Select All button. To properly resolve all dependencies, check

[x] Contact all update sites during install to find required software

Click Next, agree to the license terms, and install.

Installation Types

Depending on how close to the bleeding edge you like to be, there are several types of releases available:

  • Stable releases are - as indicated by their name - stable.
  • Development builds, released once per milestone and only a few times a year, are fairly stable, but there may be some things which do not yet work. If you would like to try one of these milestones, we'd greatly appreciate the assistance in testing and reporting of issues in our issue tracker. Please note that Update Sites and Downloads for Development Milestones will be available later this year.
  • Nightly builds contain the latest and greatest new features, but nothing is stable or guaranteed - yet. If you're using a Milestone and need a fix, you can update to the latest Nightly, or wait for the next Milestone.

Are you looking for an older version ? Look at the archives.
